# 清空按钮功能设计

## 概述

在推荐岗位和岗位列表面板中添加"清空"按钮，允许用户快速清空列表内容。

## 功能需求

### 1. 推荐岗位清空

**位置**：推荐岗位面板标题栏，与"刷新推荐岗位"按钮并排

**按钮样式**：
- Ghost样式（透明背景，边框）
- 包含删除图标（🗑️）
- 文本："清空"

**交互流程**：
1. 用户点击"清空"按钮
2. 弹出确认对话框："确定要清空所有推荐岗位吗？此操作不可撤销。"
3. 用户确认后，调用后端API清空推荐岗位
4. 前端刷新列表，显示空状态

### 2. 岗位列表清空

**位置**：岗位列表面板标题栏

**按钮样式**：
- 与推荐岗位清空按钮保持一致

**交互流程**：
1. 用户点击"清空"按钮
2. 弹出确认对话框："确定要清空所有岗位吗？此操作不可撤销。"
3. 用户确认后，调用后端API清空岗位列表
4. 前端刷新列表，显示空状态

## 技术实现

### 前端改动

**文件**：`frontend/src/App.jsx`

**推荐岗位清空**：
- 在第822-825行（推荐岗位标题栏）添加清空按钮
- 添加点击事件处理函数 `handleClearRecommendations`
- 使用 `confirm()` 进行确认

**岗位列表清空**：
- 在第1084行（岗位列表标题）添加清空按钮
- 添加点击事件处理函数 `handleClearJobs`
- 使用 `confirm()` 进行确认

### 后端API

**推荐岗位清空**：
- 端点：`POST /recommendations/clear`
- 功能：将所有非dismissed的推荐标记为dismissed
- 返回：`{"ok": true}`

**岗位列表清空**：
- 端点：`POST /jobs/clear`
- 功能：删除所有岗位记录
- 返回：`{"ok": true}`

### API客户端

**文件**：`frontend/src/lib/api.js`

添加两个新函数：
- `clearRecommendations()` - 调用推荐岗位清空API
- `clearJobs()` - 调用岗位列表清空API

## 样式参考

使用现有的ghost样式类，与"刷新推荐岗位"按钮保持视觉一致。

## 测试场景

1. 点击清空按钮，确认对话框正常显示
2. 确认清空后，列表正确清空
3. 取消清空后，列表保持不变
4. 清空后显示正确的空状态提示
